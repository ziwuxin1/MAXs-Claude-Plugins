# 麦壳思「图生图·多视图参考」SOP

适用范围:麦壳思全部"一张基准图出一组视角"的 AI 技术美术工作——AI 全流程 UE5 课程的参考板单元、学生 PureRef 作业、3D 建模前的施工图参考、Hyper3D 重建喂图、付费课程演示素材。

本 skill 是三件套的第三件:`maxs-asset-breakdown` 管"从零写提示词"，`maxs-image-to-prompt` 管"从图反推+二创"，本 skill 管"基准图 → 多视图"。基准图本身怎么来，用前两个 skill；基准图到手之后，从这里开始。

## 内核一句话

**多视图不是重新生成，是"同一个东西换镜头拍"。** 场景身份(风格/配色/结构)整段锁死，唯一在动的是镜头。每张新图都要让人一眼认出"这还是那个建筑"，又要真的拍平到能当图纸量尺寸。

## 工具事实(硬约束，写错全盘崩)

针对 Nano Banana / Banana Pro / Nano Banana 2(都是 Gemini 系，TAPNOW 同源)，以下是验证过的模型事实。本 SOP 的提示词在三者间通用:

| 事实 | 后果 |
|------|------|
| **不支持 seed 参数** | 别教、别写、别试"记下种子复用"。一致性靠**参考图 + 同一对话** |
| **没有负面提示参数**(--no / negativePrompt 无效) | 排除项写成自然语言句子收尾:"纯白背景。无阴影。无水印" |
| **对话式语义模型，不吃关键词堆叠** | 用完整中文句子下指令，比英文 tag 列表稳得多 |
| **hex 色值有效，但要绑物件写** | "朱红柱 #8B2500"有效；单独堆一排色值会被乱分配 |
| **同一对话内有上下文记忆** | 所有角度在同一个对话里连续生成——上下文就是你的"种子" |
| **香蕉 2 快 3-5 倍、约 95% 画质，中文理解更强** | 抽卡阶段用香蕉 2 控成本，选中的图细节不够再交给 Pro 精修 |

## 第一原则:三个永远要回到的判断标准

每生成一张视图，回到这三个问题:

1. **认得出是同一个东西吗** —— 新视角的图和基准图放在一起，风格、配色、关键结构(屋顶形制、开间数、标志物)必须一眼对得上。认不出就不是多视图，是另一个建筑。
2. **真的拍平了吗** —— 正面/侧面/俯视图必须无近大远小:平行线不收拢、柱子不汇聚、屋檐不变形。带透视的"正面图"当不了图纸。
3. **能当施工图用吗** —— 建模的人要能在图上量比例、描轮廓。物体完整入画、不被裁切、背景干净、细节级别和基准图一致。

## 黄金公式

```
上传基准图 + [保持不变:风格/配色/结构] + [视角指令 ← 只改这里] + [拍平无近大远小]
```

落成完整句子的标准模板:

> "参考这张图，保持建筑的风格、配色和结构完全不变，生成它的**左侧正视图**——镜头垂直对着建筑侧面，拍平、没有近大远小，建筑完整居中，背景保持纯色。"

**两个不能动的骨架**:"保持……完全不变"的保持句每个角度原样复制；"拍平、没有近大远小"是正交性的人话表达，正交视角(正/侧/俯)必带。
**一个唯一变量**:视角指令。全 prompt 只允许改这一段。

各视角的标准指令、变体写法和完整对话案例，见 `references/view-prompt-library.md`——给老师/学生写具体提示词时要读这个文件。

## 核心动作(按顺序执行)

### 动作 1:先定用途，再定出哪些视角

| 用途 | 出图清单 | 特殊要求 |
|------|---------|---------|
| 建模施工图 / PureRef 参考板 | 正面 + 左侧 + 背面 + 俯视(4 张拍平) + 3/4 视角 1 张 + 细节特写 2 张 | 正交 4 张必须拍平 |
| Hyper3D / 3D 重建喂图 | 3/4 视角为主，白底无阴影 | 单件完整轮廓，转 `maxs-asset-breakdown` 的白底规范 |
| 美宣 / 封面系列 | 3/4 + 低角度 + 特写，可带氛围 | 不要求拍平，转 `maxs-image-to-prompt` 模式① |

用途不明不允许开始出图——施工图要拍平素色，美宣要透视氛围，方向完全相反。

### 动作 2:定基准图

挑选标准:风格满意、结构看得清(3/4 或正面视角最佳)、没有大面积遮挡。基准图不满意就先回 `maxs-asset-breakdown`(从零生)或 `maxs-image-to-prompt`(从参考图改)把基准图做对，**不要拿着将就的基准图开始铺多视图**——基准图的每个毛病都会被复制到一组图里。

### 动作 3:同一对话里逐角度生成

- 开一个对话，上传基准图，之后**不开新对话**，所有角度在这个对话里连续要。
- 顺序推荐:正面 → 左侧 → 背面 → 俯视 → 3/4 → 特写。先做信息量大的正面，模型对这个建筑的"理解"会在对话里累积。
- 每个角度抽 2-4 张再判断。香蕉是概率模型，单张失败不代表提示词错。
- 抽 3 次还不对 → 调视角指令措辞；抽 5 次还不对 → 检查是不是踩了已知坑(见下表)。

### 动作 4:三查自检(每张图都过一遍)

1. **轮廓查**:结构完整入画、没被裁切、边缘清晰可描
2. **比例查**:和基准图比，开间数、层数、高宽比一致
3. **色彩查**:主色没漂移(对照保持句里绑定的色值)

三查通过的图才进参考板。这个清单也直接发给学生当作业自查表。

### 动作 5:输出复用包

交付不是一组图，是图 + 记录。沿用 `maxs-asset-breakdown` 的最小复用包格式:最终提示词组(保持句 + 每个视角的指令)、模型与参数、用途、迭代过程、踩坑提醒。S/A 级必须沉淀进麦壳思素材库。

## 已知坑(多视图特有)

| 坑 | 表现 | 应对 |
|----|------|------|
| **机位没真动** | 要"右前 3/4"出来的还是基准图同侧——方位词(左前/右后)模型容易犯懒 | 不说镜头说画面:**指明地标在画面哪边**(2026-06-09 屋顶残件实测:俯视图用"屋脊在画面上方水平贯穿"一次过) |
| **镜像冒充转角** | 指定的地标位置在几何上不可能时，模型直接左右翻转物体来"满足"指令——看着像新角度，其实是镜像，**喂重建必废** | 地标位置必须先在脑内核实几何可行性；横长物体优先用"近/远"描述:"断梁端面离镜头最近，翘角在画面远处"。自查:把出图和基准图并排，若布局是精确左右翻转即为镜像，弃 |
| **换视角变样** | 侧面图的屋顶形制/开间数和正面对不上 | 保持句里**点名英雄结构**:"保持庑殿顶、三开间、朱红柱完全不变"，别只说"保持结构不变" |
| **背面脑补失控** | 背面图凭空多出门窗、招牌 | 指令里加"背面结构合理延续正面逻辑，保持对称，不新增构件" |
| **俯视出成 3/4** | 要俯视图给你一张斜 45° 鸟瞰 | 强调"正上方垂直向下看，像屋顶平面图一样，不带任何透视" |
| **颜色漂移** | 换角度后朱红变橘红 | 把色值绑物件写进保持句:"朱红柱 #8B2500、瓦灰屋面 #6B6B6B 保持不变" |
| **风格漂移** | 越生成越写实/越卡通 | 确认还在同一个对话；保持句重申风格词；必要时把基准图再发一遍 |
| **细节级别不齐** | 正面密侧面糙 | 指令里加"细节密度与参考图一致"；低模参考和高模参考分两套对话生成 |

遇到坑先查表走应对，**不要堆约束**——加约束是最后手段(三步迭代的"简化"原则同样适用，见 `maxs-asset-breakdown`)。

## 分级体系

沿用 `maxs-asset-breakdown` 的 S/A/B/C 分级:

- **S 级**(课程标准案例/公开 portfolio):完整 6-8 张视图组 + 三查记录 + 复用包 + 换一个题材复跑验证 + 进素材库
- **A 级**(课程演示/提案):4 张正交 + 3/4 + 简化复用包
- **B 级**(个人探索):出几张存档即可
- **C 级**(一次性):不沉淀

对外内容、付费课程素材、学生作业模板必须 A 级以上。

## 教学场景下的额外要求

- **指令必须变量化**:给学生的是"保持句(骨架) + 视角指令(变量)"模板，不是某次成功的原文。
- **必须演示失败案例**:每个教学单元至少一个"背面脑补/俯视不平"的真实翻车图。一个失败到成功的迭代过程 > 三个成功案例。
- **必须给三查清单**:学生自己能判断哪张能用，不依赖老师的眼睛。

## 在对话中如何运用

用户带着任务来时，按这个优先级:

1. **先问用途**(施工图/重建喂图/美宣?)——决定视角清单和拍平要求
2. **确认基准图状态**——没有或不满意，先转前两个 skill
3. **套黄金公式**——读 `references/view-prompt-library.md`，按用途出整组视角指令
4. **提醒工具事实**——用户提到 seed/负面提示/英文关键词堆叠时，直接纠正并给替代方案
5. **交付复用包**——提示词组 + 三查清单，不是孤零零一段话

公司资产沉淀原则与三件套一致:验证过的指令、翻车案例、变体写法，沉淀进麦壳思素材库，不留在个人聊天记录里。
