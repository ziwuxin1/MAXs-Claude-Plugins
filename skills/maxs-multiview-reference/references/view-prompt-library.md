# 视角指令库 · Banana Pro 图生图多视图

所有指令都接在同一个保持句后面使用。先写保持句，再接视角指令，最后收"拍平"要求(正交视角)。

## 保持句(骨架，每个角度原样复制)

基础版:
> 参考这张图，保持建筑的风格、配色和结构完全不变，

加强版(出现"换视角变样"或"颜色漂移"时升级):
> 参考这张图，保持建筑的【庑殿顶、三开间、朱红立柱 #8B2500、瓦灰屋面 #6B6B6B】完全不变，保持手绘质感的半写实风格不变，

要点:加强版把**英雄结构点名** + **色值绑物件**写进保持句。【】内按实际题材替换。

## 八个视角的标准指令

| 视角 | 指令(接在保持句后) | 正交收尾 |
|------|-------------------|---------|
| 正面 | 生成它的正面正视图——镜头正对建筑正面，左右对称构图 | ✅ 拍平、没有近大远小 |
| 背面 | 生成它的背面正视图——镜头正对建筑背面，背面结构合理延续正面逻辑，保持对称，不新增构件 | ✅ |
| 左侧 | 生成它的左侧正视图——镜头垂直对着建筑左侧面 | ✅ |
| 右侧 | 生成它的右侧正视图——镜头垂直对着建筑右侧面(与左侧镜像对应) | ✅ |
| 俯视 | 生成它的正上方俯视图——镜头从正上方垂直向下看，像屋顶平面图一样，不带任何透视 | ✅ |
| 仰视 | 生成它的低角度仰视图——镜头贴近地面向上看，展示屋檐底部结构 | ❌ 允许透视 |
| 3/4 | 生成它的前左 3/4 视角图——略低角度，展示整体体积感 | ❌ 允许透视 |
| 特写 | 生成【斗拱/屋脊兽/柱础】的细节特写——细节密度与参考图一致 | ❌ |

统一收尾(所有视角通用):
> 建筑完整居中，不裁切，背景保持纯色。

## 完整对话流程示例(明代庭院寺庙 · 施工图用途)

**第 0 步**:开新对话，上传基准图(3/4 视角概念图)。

**第 1 条**:
> 参考这张图，保持建筑的风格、配色和结构完全不变，生成它的正面正视图——镜头正对建筑正面，左右对称构图，拍平、没有近大远小，建筑完整居中，背景保持纯色。

**第 2 条**(同对话继续):
> 同样的要求，生成它的左侧正视图——镜头垂直对着建筑左侧面。

**第 3 条**:
> 同样的要求，生成它的背面正视图——背面结构合理延续正面逻辑，保持对称，不新增构件。

**第 4 条**:
> 同样的要求，生成它的正上方俯视图——镜头从正上方垂直向下看，像屋顶平面图一样，不带任何透视。

**第 5 条**:
> 现在生成它的前左 3/4 视角图——略低角度，展示整体体积感(这张不需要拍平)。

**第 6 条**:
> 生成屋檐斗拱部分的细节特写，细节密度与参考图一致。

要点:从第 2 条起可以用"同样的要求"承接保持句——同一对话里模型记得。但一旦发现漂移，立刻回到完整保持句(加强版)重写。

## 翻车应对速查(按症状)

- **侧面和正面对不上** → 保持句升级为加强版(点名英雄结构)
- **背面多出东西** → 背面指令必带"不新增构件"
- **俯视不平** → 加"不带任何透视"，还不行就换措辞"屋顶平面图式的正投影"
- **颜色变了** → 色值绑物件进保持句；检查是否还在同一对话
- **越来越不像** → 把基准图重新上传一遍，附完整加强版保持句
- **细节级别忽高忽低** → 每条指令补"细节密度与参考图一致"；低模/高模分两套对话

## 学生作业三查清单(随作业模板发放)

每张图自查，三项全过才进 PureRef:

- [ ] 轮廓查:建筑完整入画、没裁切、边缘清晰可描
- [ ] 比例查:开间数、层数、高宽比和基准图一致
- [ ] 色彩查:主色和基准图一致，没有漂移

## Hyper3D 重建喂图:方向槽位对照

Hyper3D(Rodin)上传多图时要给每张图标方向(Front / Front Left / Back / UP 等 11 个槽位)。出图时就按槽位规划，省得回头对不上:

| 生成顺序 | 视角指令 | Hyper3D 标注 |
|---------|---------|-------------|
| 基准图 | (通常是左前 3/4) | Front Left |
| 1 | 右前方 3/4——镜头绕到另一侧，相同俯角 | Front Right |
| 2 | 正后方 3/4——背面合理延续正面逻辑，不新增构件 | Back |
| 3 | 左后方 3/4——相同俯角 | Back Left |
| 4 | 正上方俯视——垂直向下，不带透视 | UP |

要点:
- **重建喂图不要拍平**(和施工图相反)——Rodin 需要体积信息，保持同一俯角绕圈拍。
- 4-6 张即可，不必填满 11 个槽位；底面(Down)一般不补，模型会自动收底。
- 方向拿不准就标 Unknown——**标错方向比不标更伤重建**。
- 残损类资产(破洞、缺口)的"指纹"必须写进保持句:破洞位置和大小、瓦垄数等。换角度后指纹对不上，Rodin 拼不回同一个物体。

## TAPNOW"摄影机控制"面板的正确用法

该面板是镜头模拟(机型/焦距/光圈)，**不能控制机位方向**——转视角仍靠提示词指明地标位置。间接用法:

- 焦距选 85-135mm:长焦透视收敛弱，更接近"拍平"
- 光圈 f/8 以上:f/1.4 浅景深会糊掉物体边缘，重建喂图最忌讳轮廓发虚
- 拿不准就整个关掉，让提示词全权控制

## 变体:非建筑题材

题材换成载具/道具/角色时，公式不变，只换名词:

- 载具:"保持涂装、结构、磨损细节完全不变，生成它的左侧正视图——镜头垂直对着车身侧面"
- 道具(单件):优先走 `maxs-asset-breakdown` 白底规范，再用本公式换角度
- 角色:正交概念弱化为"T-pose 三视图"，加"保持服装、配色、身材比例完全不变"
