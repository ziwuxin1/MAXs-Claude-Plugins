## 填写示例(雪山古建白底图)

```markdown
# 黑神话风格雪山古建白底资产陈列图

**创建日期**: 2026-03-09  
**作者**: 张君亮  
**分级**: S  
**关联项目**: AI 全流程 UE5 场景课程 - 国风资产篇

## 最终提示词

\`\`\`
把图中的雪山古建场景结构部件按照3D资产图一次摆放,
20个结构部件不能重复,按照大中小依次排放,
8k分辨率,顶级摄影照片,俯视35度视角,
电影级布光,黄昏暖色侧光搭配冷色环境光,
风格化PBR材质,半写实搭配法线贴图层次,
辅以手绘质感的磨损、积雪、冰挂、风化细节,
保持物件比例关系准确,
黑神话悟空风格,asset showcase陈列图,白背景。无阴影
\`\`\`

## 模型与参数

| 项目 | 值 |
|------|------|
| 模型 | Banana Pro |
| 比例 | 16:9 |
| 输入图 | 雪山古建场景概念图(image-to-image) |

## 用途

- **主用途**: 教学素材 - AI 全流程 UE5 课程国风篇
- **次用途**: 学生作业模板、MAXS portfolio 展示
- **预计使用场景**: 国风资产章节第 2 节 - 资产拆解工作流

## 迭代过程

### 第一版(试水)
- 没提背景,出来深色氛围
- 学到: 不写背景指令,模型会按光照氛围脑补

### 第二版(纠错)  
- 加了 RGB(255,255,255) + 无影灯 + 产品摄影
- 问题: 物件被漂白失去原色,出现幽灵半成品 bug

### 第三版(定型)
- 删掉所有冗余精确指令,只用"白背景。无阴影"放最后
- 关键简化点: 信任模型对短指令的理解,不要堆砌
- 为什么这版最稳: 三个骨架句到位,光照不改动,只在结尾追加背景指令

## 踩坑提醒

1. **不要叠加 RGB 值和"产品摄影"等术语**,会触发过度规训
2. **黄昏光不污染白底** —— 反直觉但已验证 3 次
3. **`白背景。无阴影` 必须放最后**,放中间效果差
4. **数量给 20 件实际会出 25 件左右**,这是 Banana Pro 特性

## 可复用范围

### 已验证可替换的变量
- [x] 场景类型: 雪山古建、街角咖啡馆已验证
- [x] 数量: 15-20 件最稳
- [x] 光照: 黄昏侧光、月夜冷光已验证
- [x] 风格: 黑神话悟空、赛博朋克 2077 已验证

### 不能动的骨架
- `按照大中小依次排放`
- `保持物件比例关系准确`
- `白背景。无阴影`(必须放最后)

### 不适用的场景
- 不适用于单件展示(用模板 2)
- 不适用于深色美宣图(用模板 3)
- 不适用于纯写实摄影风(模型会强化风格化)
```

## 入库流程

1. 填写复用包模板 → 自查所有字段都填了
2. 提交到 MAXS 提示词库 review(可由课程产品设计老师审核)
3. 审核通过后:
   - S 级 → 进入主模板库 `prompt-templates.md`
   - A 级 → 进入案例库 `iteration-cases.md`(如有迭代价值)
   - B 级 → 进入个人探索归档(不入主库,但保留可查)
4. 关联到具体课程/项目/cheat sheet 等产出物

## 不允许的偷工减料

- ❌ 只贴最终提示词不写迭代过程
- ❌ 只写"成功了"不写为什么成功
- ❌ 不标注模型和参数
- ❌ 不写不适用场景

每偷工减料一次,公司就少一份资产、多一份"这个提示词在他脑子里"的隐患。
