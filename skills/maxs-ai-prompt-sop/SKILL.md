# 麦壳思 AI 技术美术提示词工程 SOP

适用范围:麦壳思全部 AI 技术美术相关工作——AI 全流程 UE5 场景课程、Vibe Coding 工具、ShaderForge AI、AI 短片、学生作业指导、对外内容产出(小红书/B站/80 Level)、portfolio 资产、付费课程素材。

## 内核一句话

真正可复用的提示词不是越精确越好,而是经过"踩坑—过度纠正—简化收敛"三步迭代后,留下来的那个最简骨架。

## 第一原则:三个永远要回到的判断标准

每当在写、调、教提示词时,回到这三个问题:

1. **学生抄了能不能复现** —— 提示词写得再漂亮,如果换一个学生、换一个场景就出不来类似效果,这不是一个可教学的提示词,只是一次性灵感。麦壳思课程的提示词必须能**跨学生、跨题材、跨抽卡概率稳定复现**。
2. **是否经过迭代验证** —— 一次抽中的提示词不算定型,必须经过"先成功一次 → 换变量再成功一次 → 换模型再成功一次"的三重验证,才能进入素材库。
3. **简洁性是否到位** —— 同样的效果,**提示词每多一句都是负担**。最终模板必须比第一次成功的版本更短。冗长的提示词通常是在补丁式纠错,不是真正的可复用模板。

这三个标准不是事后复盘用的,是写提示词时就要预演的。

## 提示词工程的核心动作(按顺序执行)

### 动作 1:先明确"产出用途",再决定提示词结构

在写任何提示词之前,先回答一个问题:**这张图最终是给谁用的?**

| 用途 | 提示词导向 | 关键指标 |
|------|-----------|---------|
| 喂 Tripo/Hunyuan3D 做 3D 重建 | 单件、白底、3/4 视角、无阴影、保留完整轮廓 | 物体轮廓清晰可分割 |
| 项目美宣/portfolio/社交媒体封面 | 场景化、氛围光、可保留深色背景 | 视觉冲击力 |
| 课程教学素材/cheat sheet | 多件陈列、大中小排布、风格统一、白底 | 物件清晰可识别、可教学 |
| 概念阶段的 art bible 参考 | 风格锚定优先、构图次之 | 美术方向一致性 |
| 学生作业模板 | 提示词结构清晰、变量化、可替换 | 学生能套用 |

**用途不明的提示词不允许开始写**,因为同样是"资产拆解图",做教学用和做美宣用,提示词的"光照""背景""数量"全都是反方向的。先问用途,再开写,这是底线。

### 动作 2:用"五变量黄金模板"作为起点

麦壳思已验证的资产陈列图黄金模板:

```
把图中的[场景类型]结构部件按照3D资产图一次摆放,
[数量]个结构部件不能重复,按照大中小依次排放,
8k分辨率,顶级摄影照片,俯视35度视角,
电影级布光,[光照方案],
风格化PBR材质,半写实搭配法线贴图层次,
辅以手绘质感的[材质细节],
保持物件比例关系准确,
[目标风格],asset showcase陈列图,白背景。无阴影
```

**五个可替换变量**:

1. `[场景类型]` —— 雪山古建场景 / 赛博朋克街区 / 末日废土加油站 / 江南水乡集市 / 蒸汽朋克工坊 / 太空舱内饰 / 武侠客栈 / 日式神社
2. `[数量]` —— 推荐 15-20 件(甜蜜点);8-12 件适合单品精修;25+ 件容易出"幽灵半成品"bug
3. `[光照方案]` —— 黄昏暖色侧光搭配冷色环境光(国风最佳)/ 正午顶光搭配阴影补光(写实白模)/ 月夜冷光带霓虹边光(赛博)/ 阴天柔和散射光(北欧极简)
4. `[材质细节]` —— 磨损、积雪、冰挂、风化(雪山)/ 锈蚀、霓虹反光、油污、电线(赛博)/ 苔藓、藤蔓、潮湿、青砖斑驳(江南)/ 灰尘、裂纹、年代感、做旧(中世纪)
5. `[目标风格]` —— 黑神话悟空 / 赛博朋克2077 / 暗黑破坏神4 / 原神 / 鸣潮 / 荒野大镖客2 / 燕云十六声 / 守望先锋

**三个不能动的"骨架句"**(改了模板就崩):

- `按照大中小依次排放` —— 控制构图节奏
- `保持物件比例关系准确` —— 防止小物件被画得过大、大物件被画小
- `白背景。无阴影` —— 必须放在**最后一句**作为收尾,位置不能动到中间

完整模板库、踩坑案例、迭代过程见 `references/prompt-templates.md`,帮老师/学生套用变量时要读这个文件。

### 动作 3:按"三步迭代法"调试,不要一次想做对

绝大多数学生(和很多老师)的错误是:第一次写提示词就想一步到位、堆砌所有约束。结果反而触发模型的"过度规训"——指令冲突、效果僵硬、抽卡概率降低。

**正确的迭代节奏**:

| 步骤 | 目标 | 提示词长度 |
|------|------|-----------|
| **第一步:试水** | 看模型对题材的默认理解是什么 | 短,只写主体 + 风格 |
| **第二步:纠错** | 针对第一步暴露的问题加约束 | 中,加 3-5 句针对性指令 |
| **第三步:简化** | 砍掉非必要约束,保留最短骨架 | 短,回到能复用的简洁版 |

**反直觉的核心经验**:**最终能用的版本通常比中间过度纠正的版本更短**。这是麦壳思团队在 Nano Banana / Banana Pro 上反复验证的规律。

具体案例(雪山古建白底图迭代过程):

- 第一版(短):出图是深色背景 + 黄昏氛围,因为没提背景,模型按"黄昏光"脑补了环境
- 第二版(长):加了 `RGB(255,255,255)` `无影灯` `产品摄影` `无地面无投影无接触阴影` 一堆约束,反而过度规训,出图僵硬,物件被漂白
- 第三版(短):删掉所有冗余,只保留 `白背景。无阴影` 五个字放最后 —— 完美

完整的迭代案例库见 `references/iteration-cases.md`,帮学生调试提示词时要读这个文件。

### 动作 4:识别"模型 bug 模式",不要怪自己提示词写错

Nano Banana / Banana Pro 有一些已知的概率性 bug,这些不是提示词问题,是模型本身的局限。识别出来才能省卡省钱省时间。

**已知 bug 模式**:

| bug 类型 | 表现 | 应对策略 |
|---------|------|---------|
| **幽灵半成品 / asset bleeding** | 物件之间漂浮一个没画完的残片 | 数量从 20 降到 15;或同提示词 reroll 2-3 次 |
| **物件染色** | 物件被环境光染上颜色,失去原色 | 加 `保留物件本身原色`;光照改成"中性"或"产品级" |
| **白底变浅灰** | 出来不是纯白而是浅灰底 | 不要叠加 RGB 值,反而用 `白背景。无阴影` 收尾 |
| **比例错乱** | 小物件被画得和大物件一样大 | 加 `保持物件比例关系准确`;或分批拆解(大件一张/小件一张) |
| **风格漂移** | 中式飞檐画成日式神社 | 风格锚点写明 `中国传统XX结构,非日式XX风格` |
| **抠图融合失败** | 多个物件粘在一起 | 加 `物件之间保持清晰间距,不重叠不粘连` |

**硬规范**:遇到 bug 先确认是不是已知模式,是的话**直接走对应应对策略,不要再加冗余约束**。把"加约束"作为最后手段,不是第一反应。

### 动作 5:输出物必须带"复用记录",不允许只交付一张图

麦壳思的 AI 技术美术工作不是个人创作,是公司资产。每张产出的图必须带一份**最小复用包**,否则不算交付完成。

**最小复用包**:

```markdown
## [项目/课程名称] - [图的用途]

### 最终提示词
（贴上最后用的那版）

### 模型 / 参数
- 模型: Banana Pro / Nano Banana / Midjourney v6 / ...
- 比例: 16:9 / 1:1 / 9:16
- 其他参数: ...

### 用途
- 主用途: （3D重建/教学素材/美宣封面/...)
- 次用途: ...

### 迭代过程
- 第一版尝试: （写了什么,出了什么问题）
- 第二版纠错: （加了什么,改善了什么）
- 第三版定型: （最终砍到什么程度）

### 踩坑提醒
- (这次踩到了什么坑,下次怎么避免)

### 可复用范围
- 可替换变量: ...
- 验证过的题材: ...
```

复用包模板和填写示例见 `references/reuse-package-template.md`。

## 提示词分级体系(决定写多细)

不是所有任务都需要写完整模板,根据用途分级:

### S 级:课程标准案例 / 公开发布 portfolio
- 必须经过三步迭代
- 必须出最小复用包
- 必须验证跨变量复用性(至少换 2 个变量重做)
- 必须沉淀进麦壳思素材库

### A 级:课程演示素材 / 客户提案
- 必须经过两步迭代(试水 + 简化)
- 出简化版复用包(提示词 + 用途 + 1-2 条踩坑提醒)
- 不强制跨变量验证

### B 级:个人探索 / 灵感记录
- 可以一次成型
- 提示词存档即可
- 有惊喜效果再升级到 A 级

### C 级:一次性废弃稿
- 不沉淀
- 用完即丢

**硬规范**:对外宣传的内容、付费课程的素材、学生作业模板,**必须 A 级以上**。不允许把 C 级提示词喂给学生。

## 教学场景下的额外要求

当这套 SOP 用在课程内容创作时(给学生看的、给学生抄的、让学生套用的),除了上面所有标准,还要做到:

### 1. 提示词必须"可变量化"

给学生的提示词不能是"我那次成功的那一版原文",而是**带占位符的模板**。让学生明白哪些是骨架(不能动)、哪些是变量(可替换),这是教学和炫技的根本区别。

**错误示范(炫技)**:
> "我用这个提示词出了张完美图,你们抄: 把图中的雪山古建场景..."

**MAXs 标准(教学)**:
> "黄金模板的五个变量分别是 XXX、YYY、ZZZ...,三个骨架句不能动,你换任何题材都套这个结构"

### 2. 必须演示"失败案例"

学生最缺的不是成功案例,是**对失败模式的认知**。每个提示词教学单元都要至少包含一个"我以为这样写会更好,结果反而更差"的案例。

**教学价值排序**:
1. 一个完整的失败到成功的迭代过程 > 三个成功案例
2. 一个反直觉发现("加约束反而更差") > 一堆正向技巧
3. 一个 bug 应对策略 > 一个完美提示词

### 3. 必须给出"判断好坏"的标准

学生抄完提示词、生成完图,要能自己判断这张图能不能用。不能让学生陷入"老师生成的好看,我生成的我也不知道哪里不对"的迷茫。

**判断标准的可视化清单**:见 `references/output-quality-checklist.md`。每节 AI 课的作业都要附这个清单,让学生自查。

## 工作流硬规范

### 抽卡规范
- 同一提示词**至少抽 2 次**再判断是否成功,Nano Banana / Banana Pro 是概率性模型
- 抽 3 次都失败 → 调提示词,不要继续无脑抽
- 抽 5 次还不行 → 换模型 / 换工作流(Banana Pro 不行可能 Flux 行),不要硬刚一个模型

### 用途分离规范
- **白底图、深色陈列图、氛围概念图**是三种完全不同的产物,提示词逻辑不通用
- 想要白底就专门写白底版提示词,不要在氛围图基础上"加一句白底"试图改造,大概率失败
- 一次提示词只服务一个用途

### 沉淀规范
- 所有 S/A 级提示词必须沉淀到麦壳思提示词库,不允许停留在个人聊天记录里
- 每月统计高复用率提示词,提炼新模板加入 SOP
- 学生反复问的提示词问题,要进入"常见踩坑库"

## 在对话中如何运用这个 SOP

当老师/学生/课程策划带着具体任务来时(比如"帮我写个赛博朋克资产陈列图的提示词""我的白底图出不来""学生作业的提示词怎么设计"),按这个优先级:

1. **先问用途** —— 拆解、美宣、教学、3D重建?用途不明不允许动手
2. **匹配现有模板** —— 套黄金模板的五个变量,90% 的需求能直接搞定,不要从零写
3. **走三步迭代** —— 试水→纠错→简化,不要一步到位
4. **识别 bug 模式** —— 遇到问题先查已知 bug 表,不要怪用户写错
5. **输出复用包** —— 最后交付不是一张图 + 一段提示词,而是一份完整的复用记录

不要只复述 SOP 条款,要把 SOP 翻译成针对当前任务的具体提示词和具体行动。

## 公司资产沉淀原则

所有验证过的提示词模板、踩坑案例、bug 应对策略、变量库、教学话术,沉淀到麦壳思提示词 SOP 和素材库,**不允许停留在某个老师/学生的个人聊天记录或本地文档里**。这是公司资产,不是个人资产。

新老师入职、新学生加入、新课程开发,这些资产必须能被无缝复用。每次产出 S/A 级提示词,主动提醒"这个可以沉淀到麦壳思提示词库,后续 XXX 课/项目也能用",并给出可以直接入库的格式化记录。
