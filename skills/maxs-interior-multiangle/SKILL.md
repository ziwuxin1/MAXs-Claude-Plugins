# 麦壳思「室内场景·多机位参考」SOP

适用范围:麦壳思全部"一张室内概念图出一组机位"的工作——UE5 场景课的关卡参考板、art bible 内景组图、分镜预览、学生场景作业、portfolio 系列图。

与 `maxs-multiview-reference`(单物体多视图)的本质区别，先想清楚再动手:

| | 单物体多视图 | 室内场景多机位(本 skill) |
|--|--|--|
| 在转什么 | 转**物体**，镜头绕着它 | 转**镜头**，人在空间里走位 |
| 保留轴 | 物体结构/配色/破损指纹 | **空间身份**:柱网层数、光源逻辑、英雄物方位关系 |
| 可变轴 | 视角(正/侧/俯) | **机位语言**(定场/反打/仰拍/回廊俯瞰/特写) |
| 拍平要求 | 正交视角必须拍平 | **永远不拍平**，室内要透视和纵深 |
| 背景要求 | 纯白无阴影 | 保留全部环境氛围 |

## 内核一句话

**室内多机位是"派一个摄影师走进这个空间"。** 空间身份锁死——建筑结构、光从哪来、英雄物在哪——然后让镜头在里面走位。每张图都要让人相信"摄影师没出这个房间"。

## 工具事实(与三件套一致)

Nano Banana / Banana Pro / 香蕉2(Gemini 系，TAPNOW 同源):无 seed、无负面提示参数、对话式中文指令优于关键词堆叠、一致性靠**参考图+同一对话**、hex 色值绑物件写。抽卡建议:香蕉 2 跑量，Pro 精修。

## 第一原则:三个永远要回到的判断标准

1. **还在同一个房间吗** —— 柱网、层数、藻井/天花、地面材质、墙面内容(壁画/装饰)对得上基准图。对不上就是另一个房间，废。
2. **光的逻辑成立吗** —— 室内场景最值钱也最易翻车的锚点。光源位置是物理事实:顶光永远从顶来，烛火永远在佛坛。反打机位时主光应该变成逆光/侧逆光，如果还是顺光打在脸上，这张图的空间是假的。
3. **机位真的走到位了吗** —— 反打要看到基准图里看不到的那面墙，仰拍要看到天花结构。出来的图和基准图构图雷同 = 机位没走，重抽。

## 黄金公式

```
上传基准图 + [机位指令(前置!) ← 只改这里] + [构图差异声明] + [该机位的光源推演] + [空间身份保持句(殿后)] + "但不要复刻参考图的构图"
```

**顺序是生死线**(2026-06-10 佛殿实测校准):保持句在前会压死机位句——模型把"空间不变"理解成"画面不变"，整组图原样复刻。正确顺序:

1. **机位先行**:首句就是"把镜头移到/放到XX，看向XX"
2. **构图差异声明**:"这张图的构图必须和参考图明显不同:参考图是XX(平视全景)，这张是XX(俯视/仰视/反打)"，并点名"画面里能看到参考图中看不到的XX"
3. **光源推演**:按新机位重新推一遍光(反打=逆光剪影，仰拍=天光入镜)——光不是复制的，是按物理重新描述的
4. **空间身份保持句殿后**:"场景还是参考图里同一座XX:结构/英雄物/色调全部一致"
5. **收尾**:"但不要复刻参考图的构图"

标准模板(佛殿·回廊俯瞰机位):

> "把镜头移到二层回廊的栏杆边，从高处向下俯瞰佛坛——这张图的构图必须和参考图明显不同:参考图是地面平视，这张是高角度俯视，能看到佛坛顶面、烛火阵的排布和地面石板，观音像上半身在画面中央、与镜头几乎等高。天光从上方藻井来，烛火阵在下方形成暖色光斑。场景还是参考图里同一座佛殿:三层木回廊、粗木柱、鎏金观音立像(宝冠、上举持法器、两侧胁侍)、四周壁画、暗木+鎏金+烛火的昏暗色调全部一致，但不要复刻参考图的构图。"

机位指令库、完整佛殿案例对话、坑应对，见 `references/interior-shot-library.md`——写具体提示词时必读。

## 核心动作(按顺序执行)

### 动作 1:先定用途，再定机位清单

| 用途 | 机位清单 | 数量 |
|------|---------|------|
| UE5 搭场景参考板 | 定场全景(基准) + 反打 + 左右侧打 + 仰拍天花 + 俯瞰(二层往下) + 材质特写 2-3 张 | 7-9 张 |
| 室内 3D 重建喂图(Hyper3D) | 定场 + 反打 + 左右侧打 + **正仰视天花板 + 正俯视地板**(覆盖空间六个面)，顶/底两张主体不入画属正常，Hyper3D 标 Unknown | 6 张 |
| art bible 氛围组图 | 定场 + 轴线推近 + 1-2 个情绪机位(仰拍/逆光剪影) | 4-5 张 |
| 分镜预览 | 按剧本走位定制 | 按需 |
| 学生场景作业 | 定场 + 反打 + 仰拍 + 特写(最小可教学集) | 4 张 |

### 动作 2:解析基准图的"空间身份"

动手前先把三样东西写下来(这就是保持句的原料):

1. **结构**:几层、柱网怎么排、天花是什么(藻井/平棊/露明造)、地面材质
2. **光源清单**:每个光源的位置和色温——"顶部藻井天窗·冷白顶光""佛坛烛火阵·暖橙低位光""两侧灯台·点缀暖光"
3. **英雄物及方位**:主体是什么、面朝哪、和入口/镜头的相对关系

基准图反推得越准，后面每个机位越稳。需要完整反推时用 `maxs-image-to-prompt` 的场景 JSON schema。

### 动作 3:同一对话逐机位生成

- 同一对话、不开新对话，顺序推荐:反打 → 侧打 → 仰拍 → 俯瞰 → 特写(从"空间信息增量最大"的机位开始)
- 每个机位抽 2-4 张;构图和基准图雷同的直接弃(机位没走)
- **每条指令重新推演光源**:写"这个机位下光从哪边来"，不要让模型自己猜

### 动作 4:三查自检(室内版)

1. **空间查**:柱网/层数/天花/壁画位置和基准图一致，没有多出门窗或少掉楼层
2. **光向查**:光源方向符合该机位的物理逻辑(反打=逆光,仰拍=天光直入镜头)
3. **英雄物查**:主体造型、朝向、和环境的相对位置没变(佛像不能换手势、不能转身)

### 交付硬规范:每条提示词完整自含

给用户/学生的提示词组，**每条必须是"空间身份句+机位指令+光源推演"拼好的完整一条**，复制即用。禁止用"……""同样的要求(接上文)"这类省略式交付——实测(2026-06-10)用户只复制了空间身份句没带机位段，模型原样复刻了基准图。"同样的空间要求"只允许出现在**同一对话内连续发送**的场景里，不允许出现在交付文档里。

### 动作 5:输出复用包

沿用 `maxs-asset-breakdown` 复用包格式:空间身份句 + 各机位指令 + 光源清单 + 踩坑记录。S/A 级进麦壳思素材库。

## 已知坑(室内场景特有)

| 坑 | 表现 | 应对 |
|----|------|------|
| **反打脑补失控** | 背后那面墙基准图里没拍到，模型乱编出大窗/楼梯/第二尊佛 | 指令里**规定背面内容**:"入口那面墙延续相同的木结构和壁画风格，中央是大门，不新增结构"。背面自由度太高时，先生成侧打(可见内容多)再反打 |
| **光向不跟机位** | 反打机位佛像背对天窗，脸上却还有正面顶光 | 每条指令显式重推光源:"此机位下藻井天光成为逆光，人物/主体呈剪影边缘光" |
| **空间尺度漂移** | 柱距变宽、回廊从三层变两层 | 空间身份句点名数字:"三层回廊""八根粗木柱" |
| **英雄物变体** | 佛像手势变了、胁侍消失、烛台换样式 | 英雄物细节点名进保持句;特写机位先抽，确立细节后再抽远景 |
| **室内变半露天** | 反打时入口变成大开的室外天光，氛围全毁 | 写明"门外是昏暗的、看不清的过道/夜色，室内光仍以藻井和烛火为主" |
| **视高错乱** | 要人眼视角给了无人机视角 | 视高写死:"人站在地面的视线高度(约1.6米)"/"二层回廊栏杆边的高度" |
| **保持句压死机位句** | 整组图全是基准图复刻——长保持句在前、短机位句在后，模型把"空间不变"理解成"画面不变"(2026-06-10 佛殿实测:5 机位中 3 个原样复刻) | **机位指令前置**:首句就是"把镜头移到XX"；接着显式声明"这张图的构图必须和参考图明显不同:参考图是XX，这张是XX"；保持句压缩后殿后，并以"但不要复刻参考图的构图"收尾 |
| **构图雷同=机位没走** | 出来的还是基准图构图微调 | 同物体多视图的地标法:点名"画面里应看到基准图中看不到的XX" |

## 分级体系 / 教学要求 / 沉淀原则

与三件套一致:S/A/B/C 分级，对外与教学素材必须 A 级以上(至少 4 机位 + 简化复用包);教学时给"空间身份句(骨架)+机位指令(变量)"模板而非成功原文，必配一个"光向翻车"失败案例;验证过的机位话术和坑应对沉淀进麦壳思素材库。

## 在对话中如何运用

1. **先问用途**(UE5 参考板/art bible/分镜?)——定机位清单
2. **解析空间身份**——结构+光源清单+英雄物方位，写成保持句
3. **读 `references/interior
