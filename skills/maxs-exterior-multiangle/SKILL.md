# 麦壳思「室外场景·多机位参考」SOP

适用范围:麦壳思全部"一张室外概念图出一组机位"的工作——UE5 室外关卡参考板、废土/城镇/废墟 art bible、分镜预览、学生外景作业、portfolio 系列。

与 `maxs-interior-multiangle`(室内多机位)同源——黄金公式、机位前置原则、交付硬规范全部继承——但室外有三个本质差异，想清楚再动手:

| | 室内 | 室外(本 skill) |
|--|--|--|
| 光源 | 多光源清单(天窗+烛火等)，逐机位推演 | **单一太阳 + 影子方向**——影子是室外最硬的空间锚点，换机位必须重推影子落向 |
| 空间边界 | 墙体封闭，脑补有限 | **开放边界**——拉远/反打时模型会凭空脑补城市/山脉/海洋，必须显式锁定环境边界 |
| 机位范围 | 受层高限制 | 多出**无人机高角度**和**远景大全景**(看整体轮廓线)两类室内做不到的机位 |

## 内核一句话

**室外多机位是"同一片场地、同一个太阳"。** 地标结构和太阳方位锁死，镜头在场地里走位、升降、推拉。每张图的影子方向都要能用"太阳在那边"一句话解释通。

## 工具事实(与系列一致)

Nano Banana / Banana Pro / 香蕉2(Gemini 系，TAPNOW 同源):无 seed、无负面提示参数、对话式中文指令、一致性靠参考图+同一对话、hex 色值绑物件。香蕉 2 跑量抽卡，Pro 精修。涂鸦/招牌上的具体文字不要试图保留(复现必乱码)，只描述"有涂鸦痕迹"。

## 第一原则:三个永远要回到的判断标准

1. **还是同一片场地吗** —— 地标结构(大型机械/塔楼/杆线)的形状、相对位置、数量对得上基准图。地标错位 = 另一个场地，废。
2. **太阳还在原来那边吗** —— 基准图定下太阳方位后，每个机位的影子落向必须符合同一个太阳。反打机位原来的逆光要变顺光、影子朝向画面深处——影子不跟机位转的图，空间是假的。
3. **边界守住了吗** —— 拉远或反打后，环境延续基准图的世界观(废土就还是废土)，没有凭空冒出城市天际线/雪山/大海。

## 黄金公式(机位前置版，继承室内 skill)

```
上传基准图 + [机位指令(前置!) ← 只改这里] + [构图差异声明] + [太阳方位与影子推演] + [场地身份保持句(殿后)] + "但不要复刻参考图的构图"
```

**顺序是生死线**(室内 skill 2026-06-10 实测校准，室外同样适用):保持句在前会压死机位句，整组原样复刻。

标准模板(废料场·高角度俯瞰机位):

> "把镜头升到场地上空约 20 米，向下俯瞰整个废料场——这张图的构图必须和参考图明显不同:参考图是地面平视，这张是无人机高角度，能看到场地的整体布局:各结构的相对位置、地面散落物的分布、通道的走向。太阳仍在画面右上方向，所有结构的影子统一落向左下。场景还是参考图里同一片废料场:左侧多层锈蚀塔楼、右上方巨型黄色环形机械残骸、中央电线杆和拉索、拱形机库门洞、集装箱堆，锈橙+沙土+军绿的色调和过曝白天空全部一致，但不要复刻参考图的构图。"

每条提示词必须**完整自含**，禁止"……/同样的要求"省略式交付(继承交付硬规范)。

机位指令库、完整案例、坑应对，见 `references/exterior-shot-library.md`——写具体提示词时必读。

## 核心动作(按顺序执行)

### 动作 1:先定用途，再定机位清单

| 用途 | 机位清单 | 数量 |
|------|---------|------|
| UE5 室外关卡参考板 | 定场(基准) + 反打 + 左右侧打 + 高角度俯瞰 + 远景大全景 + 地标特写 2-3 张 | 8-10 张 |
| art bible 氛围组图 | 定场 + 1-2 个情绪机位(低角度仰拍地标/黄昏变体) | 3-4 张 |
| 室外结构 3D 重建喂图 | 围绕核心地标转角 4 张 + 高角度俯瞰(相当于物体多视图的 UP) | 5-6 张 |
| 学生外景作业 | 定场 + 反打 + 俯瞰 + 特写(最小可教学集) | 4 张 |

### 动作 2:解析基准图的"场地身份"

动手前写下三样东西(保持句的原料):

1. **地标结构清单**(带方位):"左侧:多层锈蚀塔楼(带涂鸦)；右上:巨型黄色环形机械残骸悬在场地上方；中央:锈蚀电线杆+斜拉线缆；深处:拱形机库门洞"——地标是室外的"指纹"，相对位置就是场地的身份
2. **太阳方位+影子方向**:看基准图影子定太阳("影子落向左下 → 太阳在右上")，写成一句话，每条指令按它重推
3. **环境边界+天空状态**:"四周是延续的废土荒地""天空过曝发白的正午晴天"——边界和天气都要锁

### 动作 3:同一对话逐机位生成

- 同一对话不开新对话；顺序:侧打 → 反打 → 高角度俯瞰 → 远景 → 特写(先做能看到新地标关系的侧打，给模型积累场地布局，再做脑补量大的反打/远景)
- 每机位抽 2-4 张;构图雷同直接弃
- **每条重推影子**:"此机位下太阳在画面X方，影子落向X"

### 动作 4:三查自检(室外版)

1. **地标查**:地标结构的形状、数量、相对位置和基准图一致
2. **影子查**:影子方向能用同一个太阳解释通(最容易被忽略、最暴露假空间)
3. **边界查**:环境延续世界观，没冒出基准图之外的大型结构/天际线;天气天空一致

### 动作 5:输出复用包

沿用系列复用包格式:场地身份句 + 各机位指令 + 太阳方位记录 + 踩坑记录。S/A 级进麦壳思素材库。

## 已知坑(室外特有)

| 坑 | 表现 | 应对 |
|----|------|------|
| **影子不跟机位** | 反打后太阳该在背后了，影子还朝镜头 | 每条指令显式重推:"此机位下太阳在画面左前方，结构影子朝镜头方向拉长" |
| **边界失控** | 拉远景后周围冒出现代城市/雪山 | 保持句锁边界:"四周是延续的废土荒地，地平线上没有城市和山脉" |
| **地标漂移/增殖** | 黄色机械残骸换了位置、多了一座塔 | 地标带方位写进保持句；先侧打积累布局再做远景 |
| **天气/时段漂移** | 正午变黄昏，氛围全变 | 锁天空:"过曝发白的正午晴天保持不变"(要黄昏变体时单独要，作为氛围机位) |
| **涂鸦文字乱码** | 想保留涂鸦字样，出来是乱码 | 只描述"有涂鸦痕迹"，不写具体文字(系列通用原则) |
| **尺度漂移** | 巨型残骸变小失去压迫感 | 写参照物:"环形残骸直径约等于塔楼三层高" |
| **保持句压死机位句** | 整组复刻基准图 | 机位前置公式(继承室内 skill，生死线) |

## 分级体系 / 教学要求 / 沉淀原则

与系列一致:S/A/B/C 分级;教学给"场地身份句(骨架)+机位指令(变量)"模板，必配一个"影子不跟机位"失败案例(室外最有教学价值的翻车);验证过的话术沉淀进麦壳思素材库。

## 在对话中如何运用

1. **先问用途**——定机位清单
2. **解析场地身份**——地标清单(带方位)+太阳方位+边界天空，写成保持句
3. **读 `references/exterior-shot-library.md`** 出整组机位指令，每条重推影子
4. **交付复用包**——每条完整自含
5. 用户提到 seed/负面提示/保留涂鸦文字时直接纠正
