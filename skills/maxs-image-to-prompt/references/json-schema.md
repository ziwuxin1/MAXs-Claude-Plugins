# 图 → JSON 两套 schema(场景图 / 资产陈列图)

拿到图先用 `SKILL.md` 动作 1 判断类型,再用对应 schema。两套不通用。所有字段值用**全中文**,看不清的标 `"不确定"`,不要脑补。

---

## A. 场景概念图 schema(美宣 / art bible / 关卡概念)

```json
{
  "图像类型": "场景概念图(atmosphere concept art)·非资产陈列图",
  "用途判断": "美宣 / art bible 概念参考 / 课程场景案例 / 关卡参考",
  "主体内容": "一句话说画面主要画了什么(英雄道具 + 它在干什么)",
  "场景类型": "题材 + 地点(如:赛博朋克末世·地下黑诊所)",
  "构图": {
    "镜头": "广角全景定场 / 中景 / 特写;视平线高度;是否叙事向",
    "焦点结构": "几个焦点、分别在画面什么位置(如:左1/3红光缺口 + 中右手术椅)",
    "画幅比例": "16:9 / 21:9 / 1:1 / 9:16",
    "纵深": "强/弱;前景→中景→背景分别是什么"
  },
  "光照方案": "抽到冷暖关系层面(如:冷绿雾光主调 + 左侧暖红光渗入,冷暖对冲;顶部环形光打亮主体)",
  "色调": "主色 + 点缀色 + 整体明暗(如:青绿冷底 + 血红点缀,整体压暗)",
  "材质细节": ["瓷砖剥落", "金属锈蚀", "血迹", "潮湿水汽"],
  "目标风格锚点": "参考的游戏/作品风格(如:赛博朋克2077 / 黑神话悟空 / 最后生还者)",
  "关键道具": ["按重要性排序,英雄道具放第一"],
  "氛围关键词": ["阴森", "body horror", "末世废弃"],
  "分辨率与画质": "8k / 电影级渲染 / 写实PBR / 体积光雾",
  "背景": "深色废墟环境(场景图基本是非白底)"
}
```

**字段抽取要点:**
- `光照方案` 是场景图最值钱的字段,务必抽到"冷暖对冲关系",这往往是二创时最该锁的保留轴。
- `构图.焦点结构` 决定二创时骨架怎么锁,要写清"几个焦点 + 各自位置"。
- `关键道具` 第一个就是"英雄道具",二创换题材时它是要被替换的核心。
- 不抽屏幕/招牌上的**具体文字**,只在 `关键道具` 里写"警示标识牌"这种氛围级描述。

## B. 资产陈列图 schema(白底/深底 · 多件物体 · 教学/3D重建)

```json
{
  "图像类型": "资产陈列图(asset showcase)·非场景图",
  "用途判断": "教学陈列 / 3D重建素材(喂Tripo/Hunyuan3D) / cheat sheet",
  "场景类型": "题材(对应黄金模板的 [场景类型])",
  "物件数量": 17,
  "排布方式": "大中小依次陈列 / 网格 / 单件居中",
  "视角": "俯视35度 / 正面平视 / 3-4视角",
  "光照方案": "对应黄金模板的 [光照方案]",
  "材质细节": ["对应黄金模板的 [材质细节]"],
  "目标风格锚点": "对应黄金模板的 [目标风格]",
  "背景": "纯白无阴影 / 浅灰 / 深色",
  "分辨率与画质": "8k 顶级摄影 / 风格化PBR + 法线层次"
}
```

**陈列图直接落到 `maxs-asset-breakdown` 的五变量黄金模板**,字段名是刻意对齐的:`场景类型 / 物件数量 / 光照方案 / 材质细节 / 目标风格锚点` 一一对应模板的五个变量槽。抽完 JSON 直接填模板即可,不用重写反推逻辑。
