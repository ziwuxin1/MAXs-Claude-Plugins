# 麦壳思「图转提示词·题材迁移与起点反推」SOP (v2)

适用范围:麦壳思全部"从一张参考图出发、但要做出**不存在的新图**"的工作——把好构图迁移到新题材、从网图反推起点做自己的场景、art bible 跨题材衍生、付费课程的"反推与迁移"教学单元。

## v2 重定位声明(先读这个，决定该不该用本 skill)

**一个铁律:目标图存在参考图，就走图生图；目标图不存在，才轮到本 skill 的文字提示词。**

| 你想要的 | 该用谁 | 为什么 |
|---------|--------|--------|
| 同一场景换机位/视角/氛围 | `maxs-multiview-reference`(单物体) / `maxs-interior-multiangle`(室内) / `maxs-exterior-multiangle`(室外) | 图生图挂参考图，身份和质感直接继承。**实测(2026-06-11 佛殿):用文字描述复现同一场景，氛围、材质、光照层次全面劣于原图**——文字复现是下策 |
| 把这张图的构图骨架搬到**全新题材** | **本 skill·模式 A** | 目标图在新世界里，没有参考图可挂，必须靠文字把骨架带过去 |
| 借这张网图起手，做**我自己的**场景 | **本 skill·模式 B** | 你要的不是复刻，是一条"可改的起点"——反推出结构化提示词，换成你的变量再生成 |

v1 的"同场景多机位/换氛围/系列变体"模式已全部移交多机位三件套，本 skill 不再提供。

## 内核一句话

**反推的价值不是复刻原图，是拿到一副可以装进新世界的"骨架"。** 构图、视线引导、冷暖对冲是骨架；题材、物件、材质是皮肉。骨架抽出来，皮肉随便换——这才是把别人的图变成自己的资产。

## 工具事实(与系列一致)

Nano Banana / Banana Pro / 香蕉2(Gemini 系，TAPNOW 同源):无 seed、无负面提示参数、对话式中文完整句子优于英文 tag 堆叠、hex 色值绑物件写。**本 skill 输出的提示词不挂 tag 尾巴**(不写"写实PBR,8k"这类逗号参数串)，画质要求写成自然语句:"写实质感，电影感布光"。图上具体文字不抽(复现必乱码)，只抽"有标识感"。

## 第一原则:三个永远要回到的判断标准

1. **骨架认得出吗** —— 迁移后的图和原图并排放，构图脉络(空间布局、视线引导、冷暖关系)一眼对得上。对不上就不是迁移，是另起炉灶。
2. **世界换干净了吗** —— 新题材里不能残留原题材的物件;佛殿搬去赛博机房，就不能还剩个香炉。名词要换得彻底。
3. **能不能复用** —— 骨架提取卡要能直接套到下一张参考图上。一次性看图说话不算，必须沉淀成可重复的结构。

## 核心动作(按顺序执行)

### 动作 1:确认走哪个模式(走错就浪费)

- 用户要"同场景再来几张" → **停**，转多机位三件套
- 用户要"这个构图，但换成XX世界" → **模式 A·题材迁移**
- 用户要"照这种感觉做一张我自己的" → **模式 B·起点反推**

### 动作 2:提取「骨架卡」(替代 v1 的完整 JSON)

骨架卡只有两层，全中文:

**骨架层(题材无关，迁移时原样保留):**
- 空间布局:对称/纵深/环绕…(如"中轴对称，主体在画面深处的轴线尽头，两侧结构形成纵深框景")
- 视线引导:光柱/道路/线缆把眼睛往哪带
- 冷暖对冲:哪边冷哪边暖、从哪来(如"顶部冷光垂直洒下 vs 底部暖光阵上映"——这是一张图最值钱的 DNA)
- 英雄物位置与体量感:主体占画面什么位置、多大比例、什么姿态感(不写它"是什么")
- 氛围基调:庄严/废弃/喧闹/静谧

**题材层(迁移时全部换掉):**
- 主体是什么、环境结构是什么、材质是什么、光源的物理载体是什么(天窗/霓虹/烛火/屏幕)

抽取硬规范:只写图里真实存在的，看不清标"不确定"，不要编;具体文字不抽。

### 动作 3A:模式 A·题材迁移

**迁移幅度铁律:时代/文化语境默认属于骨架，锁死。** 上传古代的，产出必须还是古代的(佛殿→道观/宗祠/陵墓地宫)；上传赛博的，产出还是赛博的。**跨时代/跨语境的大转弯(古建→废土/赛博)只有用户显式点名目标题材时才执行**，技能不得自行提议"要不要换成XX风"。180° 大转弯做出来的东西用户没法用——他的项目语境是定死的。

**香蕉系迁移的正确姿势:挂着原图迁移。** 本 skill 面向 Banana Pro / 香蕉2——它们是图生图模型，迁移时把原图作为参考图上传，构图骨架由参考图直接保证，文字提示词只干两件事:①声明"保留构图/光影/氛围骨架" ②把内容逐项替换成新名词并锁语境。纯文字迁移(不挂图)只在原图不能外传等特殊场合用，骨架保真度会下降。

挂图迁移的**定稿模板**(2026-06-11 佛殿→地宫多轮实测，用户确认此版效果最好):

> "参考这张图的构图、光影和氛围骨架——【点名要保留的骨架要点，含冷暖对冲】，全部保留——但把场景内容整体替换为【新题材】:【旧名词→新名词逐项对照，主体放清单第一项】。时代语境保持【XX】，写实质感，电影感布光。"

**抽卡纪律(比改词更重要):** 单点失败(主体没换/打光变平)先**同提示词重抽 2-3 次**——那多半是概率问题。仍然失败才上对应强化句(见已知坑表)，强化句是补丁不是默认。实测教训:每加一层强化约束，整体方差变大，"修正版"反而全面劣于原版——终版必须比中间版短，这就是三步迭代的简化收敛。

把骨架层原样保留，题材层在**同语境内**逐项替换名词，**句式不动、只换名词**:

> 原:"顶部藻井天窗的冷白天光垂直洒下笼罩佛像 vs 佛坛烛火阵的暖橙光从下方映上来"
> 迁移(赛博机房):"顶部环形冷白工业灯阵的光垂直洒下笼罩主服务器塔 vs 机柜阵列的暖橙指示灯光从下方映上来"

输出一条完整自含的中文提示词:骨架句(已换名词) + 新题材的材质氛围补充 + "写实质感，电影感布光"收尾。

### 动作 3B:模式 B·起点反推

把骨架卡翻译成一条**带【可改变量】标记**的提示词模板，明确告诉用户哪些是骨架(别动)、哪些是变量(换成你的):

> "【你的主体——如:鎏金观音立像】立于画面深处的中轴线尽头(骨架·别动)，【你的顶部光源——如:藻井天窗】的冷光垂直洒下笼罩主体，【你的底部暖光——如:烛火阵】从下方映上来形成上冷下暖对冲(骨架·别动)……"

用户填完变量就是他自己的第一版提示词，之后按 `maxs-asset-breakdown` 三步迭代法调。

### 动作 4:三步迭代验证(引用 maxs-asset-breakdown)

1. **试水**:先抽 2 张，看骨架有没有带过去(冷暖对冲最容易丢)
2. **纠错**:骨架丢了→把对应骨架句提前、写具体;原题材残留→检查名词有没有漏换
3. **简化**:砍掉没用的细节，留最短能稳定出图的骨架

### 动作 5:输出复用包

骨架卡 + 模式 + 最终提示词 + 迭代记录。S/A 级进麦壳思素材库。

## 已知坑

| 坑 | 表现 | 应对 |
|----|------|------|
| **想用文字复现原场景** | 出图氛围材质全面降级，"还没原图好"(2026-06-11 佛殿实测) | 这不是本 skill 的活——复现/变体走图生图三件套，挂参考图 |
| **180°大转弯** | 上传古建，产出突然疯狂麦克斯——时代/文化语境被当成可变项乱换 | 语境默认锁死:古代进古代出、赛博进赛博出;跨语境迁移仅当用户显式点名目标题材;技能不得自行提议跨语境方案(2026-06-11 用户规则) |
| **单点失败就改词(过度纠正)** | 主体没换/打光变平,立刻加强化句重写——结果约束越堆方差越大,"修正版"全面劣于原版(2026-06-11 多轮实测,最终用户确认原版最好) | **先重抽再改词**:同提示词重抽 2-3 次(系列抽卡规范)。仍失败才上对应补丁,且一次只加一个:主体没换→替换清单里主体提到第一项并加姿态细节;打光变平→补一句"影调保持参考图原样,不要提亮"。补丁用完即评估能否删掉——终版比中间版短 |
| **好图锁定·最小变更链(精修阶段)** | 已经抽到一张很满意的图,想再改一处,却回到原图重跑全量提示词——把抽中的成果重新扔回赌桌 | 把满意的图定为**新基准图**,只发最小变更指令:"只做一件事:把XX换成XX,除此之外一切原样不动"。大迁移=定稿模板一次直出+重抽;局部精修=最小变更链 |
| **冷暖对冲丢失** | 迁移图只剩单一色调，张力没了 | 对冲句是骨架层第一优先级，写具体("上冷下暖")、放靠前 |
| **原题材残留** | 赛博机房里立着香炉 | 题材层逐项过一遍，名词换彻底 |
| **骨架太松** | 新图认不出和原图的关系 | 骨架句句式原样保留，只换名词，别重写 |
| **文字复现招牌** | 乱码 | 只写"有标识感"，不写具体文字 |
| **tag 尾巴** | "写实PBR,8k,16:9"这类参数串(v1 残留) | 香蕉是语义模型，画质写成自然句;比例在生图界面里选 |

## 分级体系 / 教学要求 / 沉淀原则

与系列一致:S/A/B/C 分级(S 级须换 2 个题材验证迁移性);教学给"骨架卡+变量模板"，不给成功原文;必配一个"原题材残留"或"对冲丢失"失败案例;骨架卡和迁移案例沉淀进麦壳思素材库。

## 在对话中如何运用

1. **先分流**——同场景变体请走多机位三件套，本 skill 只接"新图"需求
2. **提取骨架卡**——骨架层/题材层分开写
3. **按模式出提示词**——A 换名词，B 标变量;每条完整自含，机位/构图描述前置
4. **三步迭代**——试水→纠错→简化
5. **交付复用包**——骨架卡是公司资产
