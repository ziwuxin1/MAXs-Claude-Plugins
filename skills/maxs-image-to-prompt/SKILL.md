# 麦壳思「图转提示词·二次创作」SOP

适用范围:麦壳思全部"从一张参考图出发"的 AI 技术美术工作——把找到的参考图变成自己的提示词、基于一张图做系列变体、风格/题材迁移、给学生演示"参考图怎么变成你的东西"、art bible 概念图衍生、portfolio 系列产出、付费课程的"反推与二创"教学单元。

本 skill 是 `maxs-ai-prompt-sop` 的姊妹篇:那个管"从零写提示词",这个管"从一张图反推 + 二次创作"。反推出来的提示词,最终落到 `maxs-ai-prompt-sop` 的黄金模板和三步迭代规范上,不另起炉灶。

## 内核一句话

**二次创作不是复刻,是"提取 DNA → 锁住场景身份 → 换镜头去拍它"。** 最有价值的反推不是把原图一比一复制出来,而是抽出这个场景的身份(题材 + 构图 DNA + 冷暖对冲),然后用镜像、换机位、换视角、换尺度,产出**同一个世界的一组不同画面**。题材和场景内容守住,变的是"怎么拍它"。能复刻只证明你看懂了,能给同一个场景出一整组镜头才证明你拿到了可复用的资产。(把构图整个搬到新题材,是另一个更激进的可选模式,不是默认。)

## 第一原则:三个永远要回到的判断标准

每当用一张图反推、做二创、教学生时,回到这三个问题:

1. **认得出关系吗(保留轴守住没)** —— 二次创作出来的图,要让人"一眼能看出跟原图是一脉的"。这个"脉"就是构图骨架(保留轴)。骨架守不住,就不是二创,是另起炉灶。
2. **够不够"新"(可变轴换得开不开)** —— 如果只是把赛博换成霓虹更亮一点,那是换皮复刻,不是二创。题材/氛围这些可变轴必须换得够开,产出要能独立成立、不依赖原图也好看。
3. **能不能复用(DNA 能不能套下一张)** —— 反推得到的 JSON DNA 和"保留轴/可变轴"拆法,要能直接套到下一张参考图上。一次性看图说话不算,必须沉淀成可重复的结构。

> 实测校准(2026-05-22,赛博黑诊所原图):默认二创幅度是**保题材·保骨架·保构图,靠镜像 / 换机位 / 换视角 / 换尺度产出同场景的一组变体**——题材和场景内容守死,连构图都可以镜像或换机位,只要"还是这个场景"认得出。"换题材"(如深海版)是更激进的可选模式,不是默认。

## 核心动作(按顺序执行)

### 动作 1:先判断"图像类型",再决定用哪套结构

拿到图先分类,因为不同类型的 JSON schema 和反推模板完全不同:

| 图像类型 | 特征 | 用哪套 | 反推落到 |
|---------|------|--------|---------|
| **场景概念图** | 有环境、氛围、叙事、纵深(美宣/art bible/关卡概念) | 场景图 JSON schema | 场景型自然语言提示词 |
| **资产陈列图** | 白底或深底、多件物体大中小排布(教学/3D重建素材) | 陈列图 JSON schema | `maxs-ai-prompt-sop` 五变量黄金模板 |

**判断错了后面全错**。一张赛博手术室场景图,硬套陈列图的"物件数量/大中小排布"字段就是错的。两套 schema 见 `references/json-schema.md`。

### 动作 2:图 → JSON,提取"DNA"

按对应 schema 把图拆成结构化 JSON(全中文)。这一步是"看懂",要客观、准确,不要脑补原图没有的东西。

**抽取硬规范:**
- 只写图里**真实存在**的东西,看不清的标"不确定",不要编。
- 屏幕/招牌上的**具体文字不抽**(只抽"有警示标识感"),因为 Nano Banana/Gemini 复现不出文字,抽了也是误导。
- 光照要抽到"冷暖关系"层面(比如"左暖右冷对冲"),不只是"暗"。冷暖对冲往往是一张图最难复现、也最值钱的 DNA。

完整两套 schema 和填写示例见 `references/json-schema.md`。

### 动作 3:拆轴 —— 标出"保留轴"和"可变轴"

这是二次创作的核心。把 JSON 里的字段分成两类:

- **保留轴(锁死,不动)** = 场景身份。包括:题材、场景内容与英雄道具、冷暖对冲关系、空间布局逻辑、叙事元素(比如"侧向暖光缺口透出剪影")。
- **可变轴(放开,而且开大)** = 怎么拍这个场景:镜像翻转、机位/视角(俯仰、左右移动、推远拉近)、焦距与尺度(广角定场 ↔ 英雄道具特写)、可选的光线时间微调。

**默认拆法(已校准):题材 + 构图 DNA 锁死,靠镜像 / 换机位 / 换视角 / 换尺度产出同场景的一组变体。** 连构图都可以镜像或换机位,只要场景身份认得出。把构图搬到新题材是另一个更激进的可选模式(见动作 4 的模式③)。

四种二创模式和详细拆轴清单见 `references/derivation-modes.md`。

### 动作 4:选二次创作模式

根据需求选(默认用①):

| 模式 | 锁什么 | 换什么 | 用途 |
|------|--------|--------|------|
| **① 同场景多机位/镜像变体(默认)** | 题材 + 场景内容 + 冷暖对冲 | 镜像翻转、机位、视角、焦距/尺度(定场↔特写) | 给同一个场景/关卡出一组 beauty shot、分镜、封面+细节图 |
| **② 换氛围** | 题材 + 构图都留 | 只换氛围基调(惊悚→高冷 / 废墟→运作中 / 白天→夜晚) | 同一场景的不同状态、不同时间 |
| **③ 换题材(激进)** | 构图骨架 | 题材整个换(赛博→深海/国风/废土) | 举一反三、把一个好构图迁移到新世界 |
| **④ 系列变体** | 题材+构图+氛围都留 | 只换细节道具/微调布局 | 高度统一的成套图 |

模式选错,二创幅度就不对(太像或太散)。**默认用①**:同场景换镜头/镜像,产出一组而不是一张。

### 动作 5:生成二次创作提示词(全中文,Nano Banana / Gemini 向)

把"保留轴原样保留 + 可变轴换成新题材"组合成一条自然语言提示词。

**生成硬规范:**
- 默认模式(同场景换镜头):**场景描述整段沿用,只改机位/视角/镜像的措辞**。例如把"广角定场,略高于视平线"换成"贴近手术椅的低角度3/4近景";镜像时把"左侧墙体破开缺口"改成"右侧墙体破开缺口",其余不动。
- 换题材模式(③):保留轴的骨架句**句式不动、只换名词**(如"左侧墙体破开透出暖光的缺口,里面立着剪影人物"→"左侧船体破开透出暖光的裂口,裂口处立着剪影人物")。
- 信息过载要**做减法**。场景图信息量大,全塞进去反而糊。优先保留:英雄道具 + 环形/顶部打光 + 冷暖对冲 + 一两个标志性环境元素。次要杂物概括成"散落杂物"即可。
- 末尾统一收画质参数:`写实PBR,8k,电影级布光,比例16:9`(或按需调比例)。

### 动作 6:走三步迭代验证(引用 maxs-ai-prompt-sop)

反推/二创出来的是**初稿**,不是定稿。按 `maxs-ai-prompt-sop` 的三步迭代法验证:

1. **试水** —— 先抽 2 张,看二创幅度对不对(太像 / 太散)
2. **纠错** —— 太像就把可变轴开更大;太散就把保留轴收紧
3. **简化** —— 砍掉没用的细节,留最短能稳定出图的骨架

Nano Banana / Banana Pro 是概率模型,同提示词至少抽 2 次再判断。

### 动作 7:输出"二创复用包"

每次二创产出要带一份记录(公司资产,不是个人灵感),内容是:原图 DNA(JSON)+ 保留轴/可变轴拆法 + 用了哪个模式 + 最终提示词 + 二创幅度反馈。模板见 `references/case-ripperdoc.md` 的格式(那是第一个完整案例)。

## 已知坑(看图反推/二创特有)

| 坑 | 表现 | 应对 |
|----|------|------|
| **文字复现** | 想复现招牌/屏幕上的字,出来全是乱码 | 只描述"有警示标识感",不写死具体文字 |
| **信息过载** | 场景图细节全塞进提示词,出图糊成一团 | 做减法,只留英雄道具+打光+冷暖对冲+1-2标志元素 |
| **冷暖对冲丢失** | 二创图只剩单一色调,失去原图的张力 | 把"左X暖光 vs 右X冷光的对冲"当保留轴明确写进提示词 |
| **变体没变(默认模式)** | 出了张跟原图几乎一样的,机位/镜像没真动 | 明确写出新机位/镜像措辞(低角度特写、镜像左右、推近拉远),别只调参数 |
| **场景身份崩了** | 换题材时太放飞,认不出是同一个构图 | 回到保留轴清单,把构图骨架句原样写回,只换名词 |

## 分级体系(决定做多细)

沿用 `maxs-ai-prompt-sop` 的分级:

- **S 级(课程标准案例/公开 portfolio)**:完整 JSON + 拆轴 + 三步迭代 + 至少做出 2 个不同题材的二创 + 出复用包 + 进素材库
- **A 级(课程演示/客户提案)**:JSON + 拆轴 + 一次二创 + 简化复用包
- **B 级(个人探索)**:JSON + 一条二创提示词存档即可
- **C 级(一次性)**:不沉淀

对外内容、付费素材、学生模板必须 A 级以上。
